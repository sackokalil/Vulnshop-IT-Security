Blind SQL Injection

Security Lab: This page documents an intentionally implemented Blind SQL Injection vulnerability in VulnShop for educational and demonstration purposes.
3. Boolean-Based Blind SQL Injection
Description

Blind SQL Injection occurs when an application is vulnerable to SQL Injection but does not directly display database contents or SQL error messages.

Instead of receiving sensitive data directly, an attacker infers information from differences in the application's response.

In VulnShop, the product search functionality builds a SQL query using unsanitized user input. An attacker can inject boolean conditions and observe whether products are returned or not.

Affected Endpoint
GET /products/search?query=
Affected Route
src/routes/product_routes.py
Affected Model / Service

src/models/product.py
src/services/product_service.py
Vulnerable Code

def search_products_by_keyword(keyword):
    conn = get_db_connection()

    keyword_upper = keyword.upper()

    if (
        "AND" in keyword_upper
        or "OR" in keyword_upper
        or "SUBSTR" in keyword_upper
        or "SUBSTRING" in keyword_upper
        or "LENGTH" in keyword_upper
        or "ASCII" in keyword_upper
    ):
        query = (
            "SELECT id, name, description, price, category, image_url "
            "FROM products "
            f"WHERE id = {keyword} "
        )

        products = conn.execute(query).fetchall()

The value supplied by the user is inserted directly into the SQL statement. Because no parameterized query is used, attackers can inject SQL logic.

Manual Exploitation Scenario

1. Open the home page.

2. Use the product search field.

3. Submit a TRUE condition:

   1 OR 5=5

4. Observe that products are returned.

5. Submit a FALSE condition:

   1 AND 5=6

6. Observe that no products are returned.
Expected Vulnerable Result

Different application responses indicate that the injected SQL condition is being evaluated by the database.

If a TRUE condition returns products and a FALSE condition returns no products, the Boolean-Based Blind SQL Injection vulnerability is confirmed.

Impact
  • Database structure discovery
  • Information disclosure through inference
  • Extraction of sensitive data over multiple requests
  • Possible preparation for further SQL Injection attacks
  • Compromise of application confidentiality
Mitigation

User input must never be concatenated directly into SQL queries. The secure solution is to use parameterized queries. 


products = conn.execute("""
    SELECT id, name, description, price, category, image_url
    FROM products
    WHERE name LIKE ?
       OR description LIKE ?
       OR category LIKE ?
    ORDER BY id DESC
""", (
    f"%{keyword}%",
    f"%{keyword}%",
    f"%{keyword}%"
)).fetchall()

Parameterized queries ensure that user input is treated as data rather than executable SQL code.

Automated Exploit Demonstration

The vulnerability can also be demonstrated automatically using a Selenium script. The script opens the vulnerable search endpoint with a boolean SQL payload.

Exploit File

exploits/blind_sqli_exploit.py
Execution Command

cd exploits
python blind_sqli_exploit.py
What the Script Does
  • Starts a browser using Selenium.
  • Opens the vulnerable product search endpoint.
  • Submits a Boolean-Based SQL Injection payload.
  • Triggers the vulnerable SQL query.
  • Shows that products are returned when the injected condition is true.
  • Confirms the Blind SQL Injection vulnerability.
Example Payload

1 OR 5=5
