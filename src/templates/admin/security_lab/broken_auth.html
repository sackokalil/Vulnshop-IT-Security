Broken Authentication

Security Lab: This page documents an intentionally implemented Broken Authentication vulnerability in VulnShop for educational and demonstration purposes.
1. Broken Authentication - Insecure Password Reset
Description

Broken Authentication occurs when authentication mechanisms are implemented incorrectly and allow attackers to compromise user accounts.

In this vulnerability, the password reset function allows anyone to change the password of an existing user account by only providing the victim's email address and a new password.

The application does not require a password reset token, email verification, security question, old password, or any proof that the requester owns the account.

Affected Endpoint
POST /forgot_password
Affected Route
src/routes/auth_routes.py(forgot_password)
Affected Template
templates/auth/forgot-password.html
Affected Model / Service

src/services/user_service.py
src/models/user.py
Vulnerable Code

@forgot_password_bp.route('/forgot_password', methods=['GET', 'POST'])
def forgot_password():

    if request.method == "POST":

        email = request.form.get("email", "").strip()
        new_password = request.form.get("new_password", "")

        user = get_user_by_email(email)

        if not user:
            flash("User not found.", "danger")
            return redirect(url_for("forgot_password.forgot_password"))

        # VULNERABILITY:
        # The application changes the password only based
        # on the submitted email address.
        #
        # Missing security controls:
        # - No reset token
        # - No email confirmation
        # - No identity verification
        # - No old password check
        # - No token expiration

        update_password_by_email(email, new_password)

        create_security_event(
            event_type="BROKEN_AUTHENTICATION",
            severity="High",
            description=f"Password reset performed for {email} without verification.",
            user_id=None,
            endpoint=request.path,
            ip_address=request.remote_addr,
            user_agent=request.headers.get("User-Agent")
        )

        flash("Password changed successfully.", "success")

        return redirect(url_for("login.login_page"))

    return render_template("auth/forgot-password.html")

The application checks whether the email exists, but it does not verify whether the requester is the real owner of that email address.

Manual Exploitation Scenario

1. Open the forgot password page:

   /forgot_password

2. Enter the victim's email address:

   soul@gmail.com

3. Enter a new password:

   12345678

4. Click on "Reset Password"

5. The application changes the victim's password.

6. The user is redirected to the login page.
Expected Vulnerable Result

If the application displays Password changed successfully. and redirects to the login page without sending or verifying any reset token, the Broken Authentication vulnerability is successfully exploited.

Impact
  • Account takeover
  • Unauthorized password change
  • Unauthorized access to user accounts
  • Privilege escalation if the victim is an administrator
  • Loss of account integrity
  • Broken password recovery process
Mitigation

The application must never reset a password only based on an email address. A secure password reset process must prove that the requester has access to the registered email account. 


import secrets
from datetime import datetime, timedelta

@forgot_password_bp.route('/forgot_password', methods=['GET', 'POST'])
def forgot_password():

    if request.method == "POST":

        email = request.form.get("email", "").strip()
        user = get_user_by_email(email)

        # Do not reveal whether the email exists.
        flash("If this email exists, a reset link has been sent.", "success")

        if not user:
            return redirect(url_for("login.login_page"))

        # SECURITY FIX:
        # Generate a secure random reset token.
        reset_token = secrets.token_urlsafe(32)
        expires_at = datetime.utcnow() + timedelta(minutes=15)

        # Store the token with the user_id and expiration time.
        save_password_reset_token(
            user_id=user["id"],
            token=reset_token,
            expires_at=expires_at
        )

        # Send the reset link to the user's email address.
        send_reset_email(
            email=user["email"],
            token=reset_token
        )

        return redirect(url_for("login.login_page"))

    return render_template("auth/forgot-password.html")
Secure Password Reset Concept

1. User submits their email address.
2. Application generates a secure random token.
3. Token is stored in the database with an expiration time.
4. Reset link is sent only to the user's registered email address.
5. User opens the reset link.
6. Application verifies the token.
7. User sets a new password.
8. Token is invalidated after successful use.

Automated Exploit Demonstration

Instead of manually filling the forgot password form, the vulnerability can also be demonstrated automatically using a Selenium exploit script.

Exploit File

exploits/broken_authentication_exploit.py
Execution Command

cd exploits
python broken_auth_exploit.py
What the Script Does
  • Opens the vulnerable forgot password page.
  • Enters the victim's email address.
  • Sets a new password.
  • Submits the password reset form.
  • Waits until the application redirects to the login page.
  • Confirms that the password was changed without verification.
Target Endpoint

POST /forgot_password
Example Configuration

TARGET_EMAIL = "soul@gmail.com"

After running the exploit script, the application changes the password of soul@gmail.com and redirects the browser to the login page.

If the exploit is successful, the script prints: 


[+] Exploit successful!
