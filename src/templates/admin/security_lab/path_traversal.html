Path Traversal

Security Lab: This page documents an intentionally implemented Path Traversal vulnerability in VulnShop for educational and demonstration purposes.
1. Path Traversal - Unauthorized File Access
Description

Path Traversal is a vulnerability that occurs when an application uses a file path provided by the user without proper validation.

An attacker may manipulate the requested file path and access files outside the intended directory.

Affected Endpoint

GET /orders/<order_id>/invoice
Affected Route

src/routes/order_routes.py
Manual Demonstration

            1. Login to the application with a valid user account. for exemple : 
            Email : user@test.com
            Password : 123456

            2. Open the normal invoice URL:

            http://127.0.0.1:5000/orders/18/invoice

            3. The application downloads the normal invoice PDF.

            4. Now modify the URL by adding the file parameter:

            http://127.0.0.1:5000/orders/18/invoice?file=../secret.txt

            5. Send the modified request in the browser.

            6. If the application downloads secret.txt instead of the invoice PDF,
            the Path Traversal vulnerability is confirmed.
Expected Result

The application returns a file that was not intended to be accessible through the invoice functionality.

Automated Exploit Demonstration

Instead of manually modifying the URL in the browser, the vulnerability can also be demonstrated automatically using a Python Selenium script.

Exploit File

            exploits/path_traversal_exploit.py
Execution Command

            cd exploits
            python path_traversal_exploit.py
What the Script Does
  • Starts a supported web browser automatically.
  • Logs in using a valid user account.
  • Maintains the authenticated session.
  • Requests the invoice download endpoint.
  • Modifies the request by adding a file parameter.
  • Sends the modified request to the application.
  • Demonstrates that a file outside the invoice directory can be downloaded.
Target Endpoint

            GET /orders/<order_id>/invoice
Example Configuration

            EMAIL = "user@test.com"
            PASSWORD = "123456"

            ORDER_ID = 18

            PAYLOAD = "../secret.txt"
Expected Result

After successful authentication, the script requests: 


            /orders/1/invoice?file=../secret.txt

If the application downloads secret.txt instead of the invoice PDF, the Path Traversal vulnerability has been successfully demonstrated.

The browser download folder should contain the requested file, showing that the application allowed access to a file outside the intended invoice directory.

Impact
  • Unauthorized file access
  • Exposure of sensitive information
  • Disclosure of internal application files
  • Violation of confidentiality
Mitigation
  • Validate all file paths received from users.
  • Restrict file access to dedicated directories.
  • Reject directory traversal sequences.
  • Use allow-lists for downloadable files.
