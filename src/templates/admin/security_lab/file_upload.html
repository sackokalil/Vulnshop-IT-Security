Unrestricted File Upload

Security Lab: This page documents an intentionally implemented unrestricted file upload vulnerability in VulnShop for educational and demonstration purposes.
1. File Upload - Malicious Profile Picture Upload
Description

Unrestricted File Upload occurs when an application allows users to upload files without properly validating the file extension, MIME type, file content, or storage location.

In this vulnerability, a normal user can upload a profile picture, but the application does not verify whether the uploaded file is really an image.

As a result, a user can upload a malicious HTML file instead of an image. The file is stored inside the public static upload folder and can later be accessed directly through the browser.

Affected Endpoint
POST /profile/upload-picture
Public Uploaded File Location
GET /static/uploads/profiles/<filename>
Affected Route
src/routes/profile_route.py
Affected Template
templates/shop/profile.html
Affected Model / Service

src/models/user.py
src/services/profile_service.py
Vulnerable Code

def save_profile_picture(image_file):
    if not image_file or image_file.filename == "":
        return None

    # VULNERABILITY:
    # The uploaded file is saved without validating:
    # - file extension
    # - MIME type
    # - real file content
    # - dangerous file names
    #
    # This allows non-image files such as .html, .js, .svg,
    # .bat, .php, or .exe to be uploaded.

    filename = secure_filename(image_file.filename)

    upload_folder = os.path.join(
        current_app.root_path,
        "static",
        "uploads",
        "profiles"
    )

    os.makedirs(upload_folder, exist_ok=True)

    file_path = os.path.join(upload_folder, filename)
    image_file.save(file_path)

    return f"uploads/profiles/{filename}"

The application stores the uploaded file inside a public static directory. Because the file is publicly reachable, the browser can open and render it directly.

Manual Exploitation Scenario

1. Login as a normal user.

2. Open the profile page:

   /profile/

3. Upload a file named:

   malicious_upload.html (for exemple the file exploits/malicious_upload.html)

4. The application saves the uploaded file in:

   static/uploads/profiles/malicious_upload.html

5. Open the uploaded file directly in the browser:

   http://127.0.0.1:5000/static/uploads/profiles/malicious_upload.html

6. The uploaded HTML file is rendered by the browser.
Example Malicious File

<!DOCTYPE html>
<html>
<head>
    <title>Malicious Upload</title>
</head>
<body>
    <h1>Unrestricted File Upload Exploited</h1>

    <p>
        This HTML file was uploaded through the profile picture upload feature.
    </p>

    <script>
        alert("Unrestricted File Upload executed!");
    </script>
</body>
</html>
Expected Vulnerable Result

If the uploaded file can be opened through /static/uploads/profiles/malicious_upload.html and the browser executes its HTML or JavaScript content, the unrestricted file upload vulnerability is confirmed.

Impact
  • Upload of non-image files through a profile picture feature
  • Public hosting of malicious files
  • Stored XSS through uploaded HTML or SVG files
  • Malware or phishing file hosting
  • Abuse of server storage
  • Loss of control over uploaded user content
Mitigation

The application must strictly validate uploaded files before storing them and must avoid serving untrusted uploaded files as executable or renderable content. 


ALLOWED_EXTENSIONS = {"png", "jpg", "jpeg", "webp"}
ALLOWED_MIME_TYPES = {"image/png", "image/jpeg", "image/webp"}

def allowed_file(filename):
    return (
        "." in filename
        and filename.rsplit(".", 1)[1].lower() in ALLOWED_EXTENSIONS
    )

def save_profile_picture_secure(image_file):
    if not image_file or image_file.filename == "":
        return None

    filename = secure_filename(image_file.filename)

    # SECURITY FIX:
    # Check file extension.
    if not allowed_file(filename):
        abort(400)

    # SECURITY FIX:
    # Check MIME type.
    if image_file.mimetype not in ALLOWED_MIME_TYPES:
        abort(400)

    # SECURITY FIX:
    # Generate a new server-side filename.
    extension = filename.rsplit(".", 1)[1].lower()
    safe_filename = f"{uuid.uuid4()}.{extension}"

    upload_folder = os.path.join(
        current_app.root_path,
        "static",
        "uploads",
        "profiles"
    )

    os.makedirs(upload_folder, exist_ok=True)

    file_path = os.path.join(upload_folder, safe_filename)
    image_file.save(file_path)

    return f"uploads/profiles/{safe_filename}"
Additional Security Recommendations
  • Validate both file extension and MIME type.
  • Verify the real file content using image processing libraries.
  • Rename uploaded files with random server-generated names.
  • Store uploads outside publicly executable directories when possible.
  • Disable execution of scripts in upload directories.
  • Limit file size.
  • Serve uploaded files with safe headers such as Content-Disposition: attachment.

Automated Exploit Demonstration

Instead of uploading the malicious file manually through the profile page, the vulnerability can also be demonstrated automatically using a Selenium script.

Exploit File

exploits/upload_exploit.py
Execution Command

cd exploits
python upload_exploit.py
What the Script Does
  • Starts a browser using Selenium.
  • Logs in as a normal user.
  • Creates a malicious HTML file locally.
  • Uploads the file through the profile picture upload form.
  • Opens the uploaded file from the public static directory.
  • Confirms that the uploaded HTML file is accessible and executable.
Target Endpoint

POST /profile/upload-picture
Example Configuration

USER_EMAIL = "mous@gmail.com"
USER_PASSWORD = "sacko"

MALICIOUS_FILENAME = "malicious_upload.html"
Expected Result

After running the exploit script, the malicious file is uploaded to: 


http://127.0.0.1:5000/static/uploads/profiles/malicious_upload.html

If the browser opens the uploaded HTML file and displays the JavaScript alert, the file upload vulnerability is successfully exploited.
